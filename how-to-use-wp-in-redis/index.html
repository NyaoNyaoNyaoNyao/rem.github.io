如何在Wordpress中使用redis?

本篇简要纯文字教程主要面对Wordpress的小白用户，使用1panel面板作为基础，一学就会，一做就对。#Redis Object Cache Pro插件

首先，这里明确需要一台自己的vps。我们使用示例为1panel 社区版本进行演示。安装Github版本1panel面板等基础操作不再赘述，详情自行查看Github官方教程。

我们需要在1panel应用商店安装openresty（关于openresty开启http3详情查看）php8.2或以上（1panel面板中采取“运行环境”的方式，记得拓展模板这里选择Wordpress)，redis（推荐使用redis的衍生高性能版本keydb）

上述实现完成后，在容器这里设置端口为：

127.0.0.1 -- xxxx（安装后的默认端口 -- tcp/udp（可选，若要添加udp则另起一行重新输入，最后选择“udp”即可）

添加网站（设置自己拥有的域名例如 blog.example.com ，添加ssl证书，上传/复制ssl证书，打开HTS和取消勾选除了tls1.3之外的选项。

进入网站文件夹，找到/sites/index，删除/index文件夹下的所有内容，然后远程从Wordpress官网下载 Wordpress到vps并解压缩，将解压出来的Wordpress文件全部移动到/index文件夹下，然后返回上级目录，点击编辑权限，编辑权限“用户组”和“公共”并设置为“1000”。此步骤完成。

然后是正常安装Wordpress，详情不再赘述。下面是如何使用Redis Object Cache Pro的重点，在/index（安装Wordpress的文件夹里）/wp_config，进入，在最后那里插入以下代码：

define('WP_REDIS_CONFIG', [
'token' => 'XXXXXXXXXXXXXXXXXXXXXXXXX', // 插件的许可证
'host' => '1Panel-keydb-7FpI', //1panel中redis的容器名，按照你自己的填写，千万不要照抄
'password' => 'XXXXXXXXX', //按照你自己的填写，千万不要照抄，如果没有密码则本行全行删除
'port' => 6379,
'database' => 0, // 一般情况下使用
'timeout' => 0.5,
'read_timeout' => 0.5,
'retry_interval' => 10,
'retries' => 3,
'backoff' => 'smart',
'async_flush' => true, // 数据异步刷新
'split_alloptions' => true,
'prefetch' => true,
'strict' => true,
'debug' => false,
'save_commands' => false,
'maxttl' => 3600 * 24 * 7, // 缓存生存时间: 7x24 小时,如果30天就把7换成30
]);
define('WP_REDIS_DISABLED', getenv('WP_REDIS_DISABLED') ?: false);

然后我们保存，重载openresty，返回Wordpress，点击插件的“Enable”后，没有报错即成功。

 

自此，你完成了在Wordpress上使用Redis的操作，祝你好运~~~
